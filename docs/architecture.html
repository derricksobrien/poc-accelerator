System Overview

POC Accelerator is a Retrieval-Augmented Generation (RAG) system that transforms unstructured GitHub repository data into structured proof-of-concept instructions.

Core Modules

üì• Module A: Data Ingestion

Purpose: Load and parse repository metadata

Key Class: CatalogScraper

Input: catalog.json with 50+ accelerators

Output: Structured CatalogItem objects with validation

Loads Microsoft Solution Accelerators with full metadata validation via Pydantic

‚úîÔ∏è Module B: Data Validation

Purpose: Ensure schema compliance and type safety

Key Classes: CatalogItem, ContextBlock

Pattern: Pydantic models with enum-based categories

Provides type-safe data handling with enum-based categories (SolutionAreaEnum, ComplexityLevel)

üîç Module C: Vector Store

Purpose: Index metadata for semantic search

Key Class: SimpleVectorStore

Features: TF-IDF token matching with cosine similarity ranking

Lightweight in-memory store with filtering by solution_area + complexity_level

üì° Module D: REST API

Purpose: Expose endpoints for downstream agents

Framework: FastAPI with full Swagger documentation

Endpoints: 11 production-ready endpoints

Provides JSON responses with XML tagging for token-efficient parsing

üõ°Ô∏è Module E: Governance Guardrails

Purpose: Inject Responsible AI disclaimers

Pattern: Check responsible_ai_tag and conditionally append RAI prompt

Ensures all AI-category solutions include appropriate disclaimers

Technology Stack

  • Backend: Python 3.10+, FastAPI, Flask
  • Frontend: HTML5, CSS3, JavaScript (Vanilla & Streamlit)
  • Database: PostgreSQL (caching), JSON (config)
  • AI Services: Azure AI Foundry, Azure OpenAI API
  • Deployment: Docker, Azure Container Apps, App Service
  • Testing: pytest, 100% coverage

Performance Characteristics

  • Response Time: <100ms (semantic search)

  • Memory Usage: <200MB idle, <500MB loaded
  • Startup Time: <5 seconds
