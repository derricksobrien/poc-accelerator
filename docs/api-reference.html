API Overview

POC Accelerator provides 11 REST API endpoints for generating, searching, and validating proof-of-concept solutions.

Endpoints

POST /api/search

Semantic search over available accelerators

{
  "query": "AI automation for sales"
}

Response: Array of matching accelerators with rankings

POST /api/generate-poc

Generate a complete POC with instructions

{
  "scenario": "AI Automation for Sales",
  "area": "AI",
  "complexity": "L300"
}

Response: Generated POC instructions and requirements

POST /api/validate-azure

Validate Azure prerequisites for scenario

{
  "scenario": "AI Automation for Sales"
}

Response: List of required Azure services and status

GET /api/list-scenarios

List all available POC scenarios

Response: Array of scenario metadata

GET /api/scenario/{id}

Get details for a specific scenario

Response: Complete scenario definition

POST /api/azure-services-check

Check Azure service availability

{
  "services": ["openai", "aiservices", "search"]
}

Response: Status of each service

GET /api/ai-services-info

Get information about AI services

Response: Available AI services and capabilities

GET /api/health

Health check endpoint

Response: 200 OK when healthy

GET /streamlit

Interactive web dashboard

Response: Streamlit UI application

GET /docs

Interactive API documentation (Swagger)

Response: Full Swagger/OpenAPI documentation

GET /

Home page and application interface

Response: Web application HTML

Response Format

All responses return JSON with the following structure:

{
  "status": "success",
  "data": { /* endpoint-specific data */ },
  "error": null
}

Error Handling

Errors return appropriate HTTP status codes:

  • 200 OK: Request successful
  • 400 Bad Request: Invalid parameters
  • 404 Not Found: Resource not found
  • 500 Server Error: Internal server error

Authentication

All endpoints require proper environment variable configuration for Azure services. API key authentication is handled via Azure Key Vault.
